# BioF3 内容维护说明

## 新增教程

教程正文放在 `docs/` 下。常见位置：

- 基础内容：`docs/basics/`
- 单细胞模块：`docs/modules/`
- 基因组学：`docs/genomics/`
- bulk RNA-seq：`docs/bulk-rnaseq/`
- 表观组学：`docs/epigenomics/`
- 蛋白质组学：`docs/proteomics/`
- 多组学整合：`docs/integration/`

Docusaurus 侧边栏当前使用自动生成：

```ts
tutorialSidebar: [{type: 'autogenerated', dirName: '.'}]
```

所以新增 Markdown 文件后，一般会自动出现在对应目录的导航中。

## 新增博客

博客文章放在 `blog/` 下，文件名建议：

```text
YYYY-MM-DD-title.md
```

作者信息在：

```text
blog/authors.yml
```

标签信息在：

```text
blog/tags.yml
```

## 图片资源

教程图片放在：

```text
static/img/tutorial/
```

Markdown 中使用绝对站点路径引用：

```markdown
![图片说明](/img/tutorial/module02/example.png)
```

文件名建议使用小写字母、数字和连字符，避免中文、空格和特殊字符。

## 可下载脚本

可下载或展示的脚本放在：

```text
static/scripts/
```

源脚本可保留在：

```text
scripts/
```

## 发布前检查

发布前执行：

```bash
npm run build
```

如果构建通过，再按 `REMOTE_DEPLOY_GUIDE.md` 执行远程同步。
